1   What is TealKit?

TealKit is a mobile AI agent powered by the Model Context Protocol (MCP). You connect it to any compatible LLM (OpenAI, Anthropic Claude, Google Gemini, Mistral, Ollama …) and your own data sources — documents, emails, websites, files — and let the AI work through multi-step tasks automatically.

Everything runs on your device. No data is sent to TealKit servers.

Key concepts

  • Playground — interactive chat with live tool use
  • Tasks — saved, reusable automation workflows that run in the background
  • MCP Tools — capabilities the AI can call (web search, email, document search, …)
  • Data Sources — your personal data connected to the agent (email, files, websites)

2   First-Time Setup

1
Open Settings (gear icon on the home screen or tap First Step if shown). Navigate to LLM Settings.
2
Choose your AI provider — OpenAI, Anthropic, Google Gemini, Mistral, Azure OpenAI, or a local Ollama instance.
3
Enter your API key. Keys are stored in the secure keychain of your device and never leave it.
4
Pick a model, e.g. gpt-4o, claude-3-5-sonnet, or gemini-2.0-flash.
5
Tap Save LLM Settings. The home-screen buttons activate automatically.
Tip: Ollama users — enter http://localhost:11434 as the base URL and leave the API key blank.

3   Playground

The Playground is an interactive chat where you can experiment with the AI and its tools in real time.

Setting up a session

  1. Write a system prompt that describes the AI's role (or tap ✨ to auto-generate one).
  2. Select tools you want the AI to use (web search, email, documents, …).
  3. If you selected Website Search or Documents, tap Start Indexing to build the local index first. A red Stop button appears during indexing so you can cancel anytime.
  4. Optionally add an initial message — the first user message sent automatically when the session starts.
  5. Tap Start Playground. The button is disabled while indexing is active.

During chat

  • Tool calls appear as collapsible cards — tap to expand and see inputs/outputs.
  • Use the attachment button to add images or files to your message.
  • Tap Reset Chat to start a new conversation with the same setup.

4   Tasks

Tasks are saved automation workflows. Each task packages an LLM, a set of tools, a system prompt, and an initial message — ready to run with a single tap.

Creating a task

  1. Go to Tasks → + New Task.
  2. Give it a name and description.
  3. Configure the LLM (can differ from your global default).
  4. Select tools and configure each one (e.g. website URLs to index, document folder).
  5. Write the system prompt and the initial user message.
  6. Tap Save.

Running a task

  • ▶ Run — runs the task with a progress dialog showing each step.
  • Interactive — opens a chat conversation wired to this task's config.

Output files

After each run TealKit saves output files (markdown, HTML, execution log) to a {task-name}_{date-time}/ subfolder in your configured output directory. Find them under Output Files on the task detail screen.

Tip: Load an example task from the Examples button to get started quickly.

5   Data Sources

Configure data sources in Settings → Data Sources. Each source can be toggled on/off per task independently.

📧 Email (Gmail / IMAP)

Connect Gmail via OAuth or configure any IMAP server. The AI can search, read, and optionally send emails on your behalf.

🔍 Web Search

Choose from DuckDuckGo (free, no key needed), Serper.dev, or SerpAPI. Enter your API key for premium providers.

📄 Documents

Point to a local folder. TealKit indexes all documents into a local DuckDB database using hybrid semantic + keyword search. Re-index anytime with the index button.

🌐 Website Search

Add up to 3 seed URLs. TealKit crawls and indexes them locally. Index before starting the Playground or Task — you can stop crawling at any time.

📍 Location

Optionally save your GPS coordinates. They are injected into every task so queries like "weather at my location" resolve automatically. Coordinates never leave your device.

☁ Google Drive

Connect with OAuth to search and read files from your Drive.

🖥 SSH

Connect to a remote SSH server. The AI can run shell commands on the remote host.

6   Output Files

TealKit saves three files per task run to a timestamped subfolder:

  • output_log.md — the main result in Markdown
  • execution_log.md — step-by-step tool call log
  • output.html — rendered HTML version of the result

Configuring the output directory

Go to Settings → General and tap Output Directory to choose a folder. Defaults to an internal app folder if left blank.

Automatic cleanup

Use the Keep files for (days) setting (1–60 days, default 3) to automatically delete old runs. Cleanup runs on app start and every hour.

7   Settings

  • LLM Settings — AI provider, model, API key, temperature
  • Data Sources — email, search, documents, websites, location
  • External MCP Servers — connect to Smithery or custom MCP servers
  • General → Output Directory — where task output files are saved
  • General → Keep files for (days) — output retention period
  • Language — English or German
  • Theme — System / Light / Dark

8   FAQ

Does TealKit store my API keys on a server?

No. All API keys are stored exclusively in the device's OS secure keychain (iOS Keychain / Android Keystore) and never leave your device.

Can I use TealKit offline?

Local Ollama models work fully offline. Cloud AI providers (OpenAI, Anthropic, etc.) and web search require an internet connection.

How do I delete all my data?

Uninstall TealKit. All local databases, keys, and settings are removed automatically.

The indexing button is stuck / slow

Large document folders or websites with many pages take time. Tap the red Stop button to cancel at any point.

Where is the output folder on Android?

By default, files go to the internal app storage. Set a custom path in Settings → General → Output Directory to save to a visible folder (e.g. Downloads).
