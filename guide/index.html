1   What is TealKit?

TealKit is a mobile AI agent powered by the Model Context Protocol (MCP). Connect it to any compatible LLM (OpenAI, Anthropic Claude, Google Gemini, Mistral, Ollama …) and your own data sources — documents, emails, websites, files — then let the AI work through multi-step tasks automatically.

Everything runs on your device. No data is sent to TealKit servers.

Key concepts

  • Playground — interactive chat with live tool use
  • Tasks — saved, reusable automation workflows
  • MCP Tools — capabilities the AI can call (web search, email, documents …)
  • Data Sources — your personal data connected to the agent

2   Platform Notes

PlatformNotes
Android Output files default to internal app storage. Set a custom path in Settings → General → Output Directory to use a visible folder (e.g. Downloads). Background task scheduling uses WorkManager.
iOS API keys are stored in iOS Keychain. File access via the Files app requires tapping Choose Directory — the system file picker remembers the grant. Background tasks run via BGTaskScheduler.
Windows Full desktop experience with direct access to local file paths. Output files are written to any folder. SSH scripts (.ps1 / .sh) placed in the configured scripts directory are auto-discovered.

3   First-Time Setup

1
Open Settings (gear icon on the home screen or tap First Step if shown). Navigate to LLM Settings.
2
Choose your AI provider — OpenAI, Anthropic, Google Gemini, Mistral, Azure OpenAI, or a local Ollama instance.
3
Enter your API key. Keys are stored in the secure keychain of your device and never leave it.
4
Pick a model, e.g. gpt-4o, claude-3-5-sonnet, or gemini-2.0-flash.
5
Tap Save LLM Settings. The home-screen buttons activate automatically.
Tip: Ollama users — enter http://localhost:11434 as the base URL and leave the API key blank.

4   Playground

The Playground is an interactive chat where you experiment with the AI and its tools in real time.

Setting up a session

  1. Write a system prompt describing the AI's role (or tap ✨ to auto-generate).
  2. Select tools (web search, email, documents …).
  3. If you selected Website Search or Documents, tap Start Indexing to build the local index. A red Stop button appears during indexing — tap to cancel anytime.
  4. Optionally set an initial message sent automatically on start.
  5. Tap Start Playground (disabled while indexing is active).

During chat

  • Tool calls appear as collapsible cards — tap to expand inputs/outputs.
  • Use the attachment button to add images or files to your message.
  • Tap Reset Chat to start fresh with the same setup.

5   Tasks

Tasks are saved automation workflows. Each task packages an LLM, tools, a system prompt, and an initial message — ready to run with one tap.

Creating a task

  1. Go to Tasks → + New Task.
  2. Give it a name and description.
  3. Configure the LLM (can differ from your global default).
  4. Select tools and configure each one (website URLs, document folder, etc.).
  5. Write the system prompt and initial user message.
  6. Tap Save.

Execution modes

  • ▶ Run — executes the task immediately with a progress dialog.
  • Interactive / Try-Out — opens the task's configuration as a live Playground chat. Perfect for testing or refining a task interactively before scheduling it.

Cron scheduling PRO

Automate any task to run on a schedule:

  • Hourly — every hour
  • Daily — once per day at a chosen time
  • Weekly — on a chosen day and time
  • Custom cron expression — full cron syntax (e.g. 0 8 * * 1-5 for weekday mornings)
Tip: Scheduled tasks run in the background even when the app is minimised. Make sure battery optimisation settings do not kill the app.

Run statistics

Tap a completed run entry to view detailed stats:

Output format

Control the output format through your system prompt or initial message:

  • Ask for Markdown (.md), HTML, JSON, or plain text
  • TealKit saves the raw result plus a rendered HTML preview automatically

Email output PRO

Have the task result emailed automatically when done. Configure in the task's Output section:

  • Recipient — one or more email addresses
  • Subject — static text or dynamic (can reference task name / date)
  • Condition — always, only on success, or only if output is non-empty

ZIP output PRO

Bundle the full output folder (result + logs + attachments) into a single ZIP file. Combine with Email output to receive the archive by mail after every run.

Task chaining PRO

Chain multiple tasks so the output of one feeds into the next. Each link can have a condition evaluated by the LLM — e.g. "only continue if the report mentions urgent issues". Subtask mode passes the parent run's output as additional context to the child task.

Example chain: Fetch & summarise emails → If summary contains "invoice" → Run accounting task → Send result ZIP by email.
Tip: Load an example task via the Examples button to get started quickly.

6   Data Sources

Configure data sources in Settings → Data Sources. Each source can be toggled on/off per task independently.

📧 Email (Gmail / IMAP)

Connect Gmail via OAuth or configure any IMAP server. The AI can search, read, and optionally send emails.

🔍 Web Search

Choose DuckDuckGo (free, no key needed), Serper.dev, or SerpAPI. Enter your API key for premium providers.

📄 Documents

Point to one or more local folders. TealKit indexes all documents into a local DuckDB database using hybrid semantic + keyword search. See Document Search for details.

🌐 Website Search

Add up to 3 seed URLs. TealKit crawls and indexes them locally. Index before starting the Playground or Task — crawling can be stopped at any time.

📍 Location

Optionally save GPS coordinates. They are injected into every task so queries like "weather at my location" resolve automatically. Coordinates never leave your device.

☁ Google Drive

Connect via OAuth to search and read files from your Drive.

🖥 SSH

Connect to a remote SSH server. See the SSH section for full details.

External MCP servers

Connect to any MCP-compatible server from the ecosystem — e.g. via Smithery.ai. Go to Settings → External MCP Servers and add a server URL or package name. External servers extend the AI with calendar access, task managers, databases, and more.

JS mini-MCP servers COMING SOON

Write your own lightweight MCP server as a small JavaScript snippet directly in the app. The snippet runs in a sandboxed environment and can expose any custom tools to the AI — ideal for one-off integrations or private APIs.

8   Document Search

TealKit's document search uses a local DuckDB database with hybrid semantic embedding + BM25 keyword search for high-quality results — no cloud dependencies.

Adding folders

  1. Tap + Choose Directory in the Documents data source settings.
  2. Use the quick-access chips (Downloads, Documents, Desktop) to add common folders in one tap.
  3. Add multiple folders — all are indexed together into a single unified index.

File type filter

Select which file types to include using the chip selector. Supported types:

pdf docx doc txt md html htm csv json xml xlsx xls pptx ppt rtf odt ods odp epub mobi log

Use All to select every type or Reset to deselect all.

Indexing

  • Tap Start Indexing to build or refresh the index.
  • A red Stop button replaces the index button while indexing is active — tap to cancel.
  • The index is stored locally; re-index only when your files change.
Tip: For very large folders, index overnight with the device plugged in.

9   SSH

The SSH server gives the AI the ability to run shell commands on a remote host. Configure the connection in Settings → Data Sources → SSH:

  • Host, port, username, password or private key path
  • Optional default working directory

What the AI can do via SSH

  • Run arbitrary shell commands and capture their output
  • Check system status, disk usage, running processes
  • Read log files and diagnose issues
  • Start or stop services
  • Execute custom scripts stored on your device

LLM-generated scripts

Ask the AI to write and immediately execute a script. Example prompt:

Connect to the server, write a bash script that gathers CPU usage every 5 seconds
for 1 minute and saves the results to /tmp/cpu_report.txt, then show me the summary.

The AI writes the script, uploads it via SSH, executes it, and returns the full output — all in one task run.

Stored local scripts

Place your own .sh or .ps1 scripts in the Scripts Directory (set in Settings → General → Scripts Directory). The AI auto-discovers all scripts and you can reference them by name in prompts:

Run the "backup_db.sh" script on the server and tell me if it completed successfully.
Tip: Keep a small library of maintenance scripts and let the AI choose the right one based on your natural-language request.

10   Output Files

TealKit saves three files per task run to a timestamped subfolder:

  • output_log.md — the main result in Markdown
  • execution_log.md — step-by-step tool call log
  • output.html — rendered HTML version of the result

Configuring the output directory

Go to Settings → General and tap Output Directory to choose a folder. Defaults to an internal app folder if left blank.

Automatic cleanup

Use the Keep files for (days) setting (1–60 days, default 3) to automatically delete old runs. Cleanup runs on app start and every hour.

11   Settings

  • LLM Settings — AI provider, model, API key, temperature
  • Data Sources — email, search, documents, websites, location, SSH
  • External MCP Servers — connect to Smithery or custom MCP servers
  • General → Output Directory — where task output files are saved
  • General → Scripts Directory — local script library for SSH tasks
  • General → Keep files for (days) — output retention period
  • Language — English or German
  • Theme — System / Light / Dark

12   FAQ

Does TealKit store my API keys on a server?

No. All API keys are stored exclusively in the device's OS secure keychain (iOS Keychain / Android Keystore) and never leave your device.

Can I use TealKit offline?

Local Ollama models work fully offline. Cloud AI providers and web search require an internet connection.

How do I delete all my data?

Uninstall TealKit. All local databases, keys, and settings are removed automatically.

The indexing is slow or stuck

Large document folders or websites with many pages take time. Tap the red Stop button to cancel at any point.

Where is the output folder on Android?

By default, files go to internal app storage. Set a custom path in Settings → General → Output Directory to use a visible folder (e.g. Downloads).

How does task chaining work?

Each chained task starts after its predecessor finishes. An optional LLM-evaluated condition controls whether to continue the chain. The previous task's output is passed as context to the next task.

What is the difference between Run and Interactive mode?

Run executes the task autonomously and saves the result to disk. Interactive opens a full chat with the same LLM and tools, so you can guide the AI step by step — useful for exploration or debugging a task.
