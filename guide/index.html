1   What is TealKit?

TealKit is a mobile AI agent powered by the Model Context Protocol (MCP). Connect it to any compatible LLM (OpenAI, Anthropic Claude, Google Gemini, Mistral, Ollama …) and your own data sources — documents, emails, websites, files — then let the AI work through multi-step tasks automatically.

Everything runs on your device. No data is sent to TealKit servers.

Key concepts

  • Playground — interactive chat with live tool use
  • Tasks — saved, reusable automation workflows
  • MCP Tools — capabilities the AI can call (web search, email, documents …)
  • Data Sources — your personal data connected to the agent

4   Playground

The Playground is an interactive chat where you experiment with the AI and its tools in real time.

Setting up a session

  1. Write a system prompt describing the AI's role (or tap ✨ to auto-generate).
  2. Select tools (web search, email, documents …).
  3. If you selected Website Search or Documents, tap Start Indexing to build the local index. A red Stop button appears during indexing — tap to cancel anytime.
  4. Optionally set an initial message sent automatically on start.
  5. Tap Start Playground (disabled while indexing is active).

During chat

  • Tool calls appear as collapsible cards — tap to expand inputs/outputs.
  • Use the attachment button to add images or files to your message.
  • Tap Reset Chat to start fresh with the same setup.

5   Tasks

Tasks are saved automation workflows. Each task packages an LLM, tools, a system prompt, and an initial message — ready to run with one tap.

Creating a task

  1. Go to Tasks → + New Task.
  2. Give it a name and description.
  3. Configure the LLM (can differ from your global default).
  4. Select tools and configure each one (website URLs, document folder, etc.).
  5. Write the system prompt and initial user message.
  6. Tap Save.

Execution modes

  • ▶ Run — executes the task immediately with a progress dialog.
  • Interactive / Try-Out — opens the task's configuration as a live Playground chat. Perfect for testing or refining a task interactively before scheduling it.

Cron scheduling PRO

Automate any task to run on a schedule:

  • Hourly — every hour
  • Daily — once per day at a chosen time
  • Weekly — on a chosen day and time
  • Custom cron expression — full cron syntax (e.g. 0 8 * * 1-5 for weekday mornings)
Tip: Scheduled tasks run in the background even when the app is minimised. Make sure battery optimisation settings do not kill the app.

Output format

Control the output format through your system prompt or initial message:

  • Ask for Markdown (.md), HTML, JSON, or plain text
  • Ask the excel server to produce a .xlsx spreadsheet from any structured data
  • Ask the chart or mermaid servers to produce .png diagram or chart images
  • TealKit saves the raw result plus a rendered HTML preview automatically

Output channels PRO

Task results can be delivered automatically to one or more channels when a run finishes. Each channel has its own send condition and can optionally attach the output files. Global credentials are configured once in Settings → Data Sources; per-task overrides are set on the task's Output tab.

Any generated file can be attached — not just the Markdown log. This includes .xlsx spreadsheets (from the excel server), .png charts and diagrams (from chart / mermaid), .html previews, and .md result files. Enable the Include output files toggle on the task's Output tab to have all generated files forwarded to every active delivery channel.

✉ Email output PRO

Have the task result emailed automatically when done. Configure in the task's Output section:

  • Recipient — one or more email addresses
  • Subject — static text or dynamic (can reference task name / date)
  • Condition — always, only on success, or only if output is non-empty
  • Include output files — attach all generated files as email attachments: .md result, .html preview, .xlsx spreadsheets, .png charts and diagrams, and any other files the AI created during the run

💬 Slack output PRO

Post the task result to a Slack channel automatically. Creating a Slack App is free — create one at api.slack.com. Two auth modes are supported:

  • Incoming Webhook — simplest option, no bot needed. To get the URL:
    1. Go to api.slack.com/apps and create a free app (From scratch).
    2. Under Add features & functionality, click Incoming Webhooks and activate them.
    3. Click Add New Webhook to Workspace, pick a channel, and allow access.
    4. Copy the Webhook URL shown — it starts with https://hooks.slack.com/services/…
    5. Paste it into Data Sources → Slack → Webhook URL in TealKit.
    Text attachments are embedded as code blocks in the message.
  • Bot Token (xoxb-…) — enables real file uploads. Steps:
    1. In your Slack App, open the left sidebar → OAuth & Permissions.
    2. Scroll to the Scopes section → Bot Token Scopes.
    3. Click Add an OAuth Scope and add: chat:write, files:write, files:read.
    4. Scroll back up and click Install to Workspace (or Reinstall).
    5. Copy the Bot User OAuth Token that starts with xoxb-…
    6. Also invite the bot to your target channel: in Slack type /invite @YourBotName.

Per-task options on the Output tab:

  • Send condition — Always / Only on success / Only on error
  • Override channel — leave blank to use the global default channel
  • Include output files — attach all generated output files: .md result log, .html preview, .xlsx spreadsheets, .png charts & diagrams, and any other files produced during the run (or the ZIP if ZIP output is enabled). Bot Token required for native file uploads; Webhook mode embeds text content inline.
Tip: Use a Bot Token when you need output files (Excel, PNG charts, …) attached as native Slack files. A Webhook is enough for text-only notifications.

📲 WhatsApp output PRO

Send the task result to a WhatsApp number via the Meta Business Cloud API. Registration is free and includes 1,000 conversations / month at no cost. Requires a Meta Business account with a verified phone number.

  1. Create or open a Meta app at developers.facebook.com → WhatsApp Cloud API Get Started.
  2. Add the WhatsApp product and note your Phone Number ID.
  3. Generate a System User access token with whatsapp_business_messaging permission.
  4. Enter both values in Settings → Data Sources → WhatsApp.

  • Send condition — Always / Only on success / Only on error
  • Override recipient — E.164 phone number (e.g. +49170…); leave blank to use the global default
  • Include output files — all generated files (.xlsx, .png, .md, .html, …) are uploaded via the WhatsApp Media API and sent as separate document messages after the text summary. Meta Cloud API only — CallMeBot (text-only) lists file names in the message instead.

ZIP output PRO

Bundle the full output folder (result + logs + attachments) into a single ZIP file. Combine with Email output to receive the archive by mail after every run.

Task chaining PRO

Chain multiple tasks so the output of one feeds into the next. Each link can have a condition evaluated by the LLM — e.g. "only continue if the report mentions urgent issues". Subtask mode passes the parent run's output as additional context to the child task.

Example chain: Fetch & summarise emails → If summary contains "invoice" → Run accounting task → Send result ZIP by email.
Tip: Load an example task via the Examples button to get started quickly.

🔧 Custom Tools

TealKit lets you build your own tools without writing any native code. Two wizards are available — accessible from the Tools tab in Playground settings or when editing a Task.

📜 Shell Script Wizard

Describe what you need in plain language and the AI writes a ready-to-use shell script (.sh / .ps1). Saved scripts are auto-discovered by the SSH server and can be referenced by name in any task.

Tips for a good shell script prompt:

  • Name the target environment: “Bash on Ubuntu”, “PowerShell on Windows Server”
  • Describe inputs and outputs: “takes a folder path as argument, writes CSV to stdout”
  • Mention error handling: “exit with code 1 on failure, print error to stderr”
  • Example: “List files modified in the last 24 hours in a given directory and save the result to report.txt”
Tip: After generating, tap Test Run to execute the script via SSH and verify the output before saving.

🧩 JavaScript Tool Wizard

Write a custom MCP tool as a small JavaScript snippet. The snippet runs in a secure on-device sandbox and can call REST APIs with fetch(). Saved tools appear immediately in the tool selector for Playground and Tasks.

Tips for a good JavaScript tool prompt:

  • Define input schema and output clearly: “input: { from, to, amount }, output: { rate, converted }”
  • For REST APIs include the full URL pattern: “GET https://api.example.com/v1/data?key={param}”
  • Mention error cases: “return an error message if the currency code is invalid”
  • Example: “Fetch the current Bitcoin price in EUR from the Coinbase public API and return price and timestamp”
Tip: Use the Browse Example Prompts button in the JS Tool editor to load working examples (calculator, currency converter, city geocoding) and see the expected tool structure.

8   Document Search

TealKit's document search uses a local DuckDB database with hybrid semantic embedding + BM25 keyword search for high-quality results — no cloud dependencies.

Adding folders

  1. Tap + Choose Directory in the Documents data source settings.
  2. Use the quick-access chips (Downloads, Documents, Desktop) to add common folders in one tap.
  3. Add multiple folders — all are indexed together into a single unified index.

File type filter

Select which file types to include using the chip selector. Supported types:

pdf docx doc txt md html htm csv json xml xlsx xls pptx ppt rtf odt ods odp epub mobi log

Use All to select every type or Reset to deselect all.

Indexing

  • Tap Start Indexing to build or refresh the index.
  • A red Stop button replaces the index button while indexing is active — tap to cancel.
  • The index is stored locally; re-index only when your files change.
Tip: For very large folders, index overnight with the device plugged in.

9   SSH

The SSH server gives the AI the ability to run shell commands on a remote host. Configure the connection in Settings → Data Sources → SSH:

  • Host, port, username, password or private key path
  • Optional default working directory

What the AI can do via SSH

  • Run arbitrary shell commands and capture their output
  • Check system status, disk usage, running processes
  • Read log files and diagnose issues
  • Start or stop services
  • Execute custom scripts stored on your device

LLM-generated scripts

Ask the AI to write and immediately execute a script. Example prompt:

Connect to the server, write a bash script that gathers CPU usage every 5 seconds
for 1 minute and saves the results to /tmp/cpu_report.txt, then show me the summary.

The AI writes the script, uploads it via SSH, executes it, and returns the full output — all in one task run.

Stored local scripts

Place your own .sh or .ps1 scripts in the Scripts Directory (set in Settings → General → Scripts Directory). The AI auto-discovers all scripts and you can reference them by name in prompts:

Run the "backup_db.sh" script on the server and tell me if it completed successfully.
Tip: Keep a small library of maintenance scripts and let the AI choose the right one based on your natural-language request.

10   Output Files

TealKit saves core files per task run to a timestamped subfolder:

  • output_log.md — the main result in Markdown
  • execution_log.md — step-by-step tool call log
  • output.html — rendered HTML version of the result

In addition, any file the AI generates during the run is saved automatically:

  • .xlsx — Excel spreadsheets created by the excel server (e.g. “Export the data as an Excel file”)
  • .png — charts generated by the chart server or Mermaid diagrams rendered by the mermaid server
  • Any other .md, .html, .csv, .json files the AI writes via the file server
Tip: Forward files to Email / Slack / WhatsApp. Enable Include output files on the task’s Output tab to have all generated files (xlsx, png, md, html …) delivered automatically via every active output channel — as email attachments, native Slack file uploads (Bot Token required), or WhatsApp document messages (Meta Cloud API).

Configuring the output directory

Go to Settings → General and tap Output Directory to choose a folder. Defaults to an internal app folder if left blank.

Automatic cleanup

Use the Keep files for (days) setting (1–60 days, default 3) to automatically delete old runs. Cleanup runs on app start and every hour.

12   FAQ

Does TealKit store my API keys on a server?

No. All API keys are stored exclusively in the device's OS secure keychain (iOS Keychain / Android Keystore) and never leave your device.

Can I use TealKit offline?

Local Ollama models work fully offline. Cloud AI providers and web search require an internet connection.

How do I delete all my data?

Uninstall TealKit. All local databases, keys, and settings are removed automatically.

The indexing is slow or stuck

Large document folders or websites with many pages take time. Tap the red Stop button to cancel at any point.

Where is the output folder on Android?

By default, files go to internal app storage. Set a custom path in Settings → General → Output Directory to use a visible folder (e.g. Downloads).

How does task chaining work?

Each chained task starts after its predecessor finishes. An optional LLM-evaluated condition controls whether to continue the chain. The previous task's output is passed as context to the next task.

What is the difference between Run and Interactive mode?

Run executes the task autonomously and saves the result to disk. Interactive opens a full chat with the same LLM and tools, so you can guide the AI step by step — useful for exploration or debugging a task.

13   JavaScript Tool PRO

The JavaScript Tool Wizard lets you create lightweight custom MCP tools using pure JavaScript — no native code or external packages required. Tools run in a secure on-device sandbox (QuickJS / JavaScriptCore) and are available immediately in Playground and Tasks after saving.

Tool structure

Every generated tool must define exactly one object named generatedTool:

const generatedTool = {
  name: "bitcoin_price",
  description: "Fetch the current Bitcoin price from CoinGecko",
  inputSchema: {
    type: "object",
    properties: {
      currency: { type: "string", description: "Target currency code, e.g. usd" }
    },
    required: ["currency"]
  },
  execute: async (args) => {
    try {
      const cur = String(args?.currency ?? "usd").toLowerCase();
      const res = await fetch(`https://api.coingecko.com/api/v3/simple/price?ids=bitcoin&vs_currencies=${cur}`);
      const data = await res.json();
      return JSON.stringify({ ok: true, price: data.bitcoin[cur], currency: cur });
    } catch (error) {
      return JSON.stringify({ ok: false, error: String(error?.message || error) });
    }
  }
};

Runtime rules & limits

  • execute(args) must return a JSON string via JSON.stringify(…) or a plain string
  • fetch() is available for HTTP/REST calls
  • console.log() output is captured and shown in test results
  • Forbidden: require, import, process, fs, path, net, child_process
  • Max script size: 64 KB  •  max response payload: 256 KB  •  default execution timeout: 8 s (max 15 s)
  • ES6 syntax only — no top-level await outside execute()

Creating a tool step by step

1
Open Playground or a Task, tap the Tools tab, then tap + New JS Tool (or JS Tool Library from Settings).
2
Type a description of what the tool should do. The AI generates the full generatedTool snippet including schema.
3
Review the generated code. Tap Test Run to execute it in the sandbox with sample arguments and see the output and any console logs.
4
Tap Save — the tool is now available in the tool selector for any Playground session or Task.
Tip: Use the Browse Example Prompts button in the editor to load working examples (calculator, currency converter, geocoding) and see the expected tool structure.

Calling saved tools via MCP

When the js_bridge server is selected, the AI can discover and call your tools automatically:

  • list_js_tools — returns all saved tools with their schemas
  • run_js_tool — call by name: { "toolName": "bitcoin_price", "args": { "currency": "eur" } }
  • Dynamic direct call — active tools are also registered as js_<name> tools (e.g. js_bitcoin_price)

Example prompts for generation

  • “Fetch today’s EUR/USD exchange rate from the free Frankfurter API and return rate and date.”
  • “Calculate compound interest. Input: principal, annualRate, years. Return: finalAmount, totalInterest.”
  • “Look up a city name using the Open-Meteo geocoding API and return latitude, longitude, and country.”
  • “Convert a Unix timestamp to a human-readable date string in a given timezone.”

14   Token & Cost Statistics

Playground — live session stats

During a Playground chat, a token counter appears at the bottom of the chat area showing the cumulative tokens consumed and characters sent in the current session. The counter turns amber when approaching the model’s configured token warning threshold.

Tap the token counter to open the full detail sheet:

15   Useful Tips

Quick practical suggestions to get the most out of TealKit.

🧩 Generate a JavaScript function

Open the JS Tool editor and describe the function precisely, including inputs, outputs, and the API to call:

Fetch the current Bitcoin price in EUR and USD from the free CoinGecko API.
Input: { currency: string }. Return: { price, currency, lastUpdated }.

Tap Generate, inspect the code, then Test Run it in the sandbox before saving. The tool is instantly available in any session.

📜 Write a shell script

Open the Shell Script Wizard and state the target environment, inputs, and expected behaviour:

Bash on Ubuntu. The script takes a folder path as argument,
finds all .log files older than 7 days, deletes them,
and prints a summary (count and freed space) to stdout.
Exit code 1 on error, error message to stderr.

After generation, tap Test Run to execute via SSH and verify the output before saving to the script library.

▶ Test a task before scheduling

Before activating a cron schedule, open the task with Interactive / Try-Out mode. This loads the task’s configuration into a live Playground chat so you can step through the AI’s tool calls and refine the system prompt — without creating any scheduled run history.

Tip: Use Interactive mode to iterate on the prompt and tool selection until the output looks right, then save the task and enable the schedule.

🌐 Register an external MCP server

Go to Settings → External MCP Servers → + Add Server and fill in:

  • Name — a friendly label for the server
  • Server URL — the HTTP(S) base URL of the MCP server, or a Smithery package name (e.g. @modelcontextprotocol/server-filesystem)
  • MCP Endpoint — typically /mcp (default); leave unchanged for Smithery
  • API Key — only if the server requires authentication

Tap Test Connection to validate the server is reachable, then Save. The server’s tools appear immediately in the tool selector.

Tip: Browse the Smithery.ai catalogue directly from the screen using the search bar — results show the package name, description, and tool count.

💰 Check current session costs

In the Playground: tap the token counter at the bottom of the chat → the detail sheet shows last-request cost and cumulative session cost.

For Tasks: open the task → tap a completed run entry → the Last Run Statistics card shows Last price and Total price.

↺ Reset a session (cleanup)

Tap the Reset Chat button (↺ icon in the toolbar, or the button below the last message) to:

  • Clear all conversation history from the current session
  • Reset the token counter and session cost to zero
  • Keep the system prompt, tools, and configuration intact

The AI starts fresh with no memory of the previous conversation — useful when switching topics or when context has grown too large.

Tip: If a long session causes the AI to lose track of earlier instructions, reset the chat and add a concise summary of what was already done as the first message.
